# SBTI CLI Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a dual-mode, offline-first `sbti-cli` package with human-friendly terminal flows, stable JSON interfaces for AI agents, bundled SBTI snapshot support, and MIT-licensed repository scaffolding.

**Architecture:** The project ships as a single npm package built with Vite for Node. Pure SBTI logic lives in `src/core`, static snapshot data and schemas live in `src/data`, command wiring and terminal output live in `src/cli`, and upstream refresh logic lives in `src/update`. Human and machine modes share the same runtime and scoring engine to prevent behavior drift.

**Tech Stack:** TypeScript, Vite, Vitest, `cac`, `@clack/prompts`, `yoctocolors`, `zod`, Node.js built-ins

## Execution Notes

- Use fixture data for unit tests and the bundled snapshot for runtime behavior.
- Keep CLI rendering thin; business logic belongs in `src/core`.
- Treat networked update logic as optional runtime behavior behind the `update` command only.
- If upstream SBTI content is bundled, keep repository code licensing and upstream attribution clearly separated.
- Do not implement prompt inference in a way that hides uncertainty or pretends to be official scoring.
